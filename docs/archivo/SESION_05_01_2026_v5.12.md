## PROBLEMA IDENTIFICADO

### Fragmentación de proveedores

**371 proveedores únicos** cuando deberían ser **~100-150**.

Ejemplos de fragmentación detectada:
- `CERES`, `CERES 2`, `2T25 0422 CERES`, `ATRASADA 1T25 0328 CERES` → Todo es **CERES**
- `BM`, `2 BM`, `BM SUPERMERCADOS 2`, `SUPERMERCADOS BM` → Todo es **BM SUPERMERCADOS**
- `VIRGEN DE LA SIERRA`, `VIRGEN DE LA SIERA` (typo), `BODEGA VIRGEN DE LA SIERRA COOP` → Todo es **VIRGEN DE LA SIERRA**

**Causa raíz:** El proveedor se extraía del nombre del archivo SIN normalizar.

## SOLUCIÓN IMPLEMENTADA (v5.12)

### 1. Normalización mejorada en `normalizar_proveedor()`

```python
# ANTES: No se aplicaba normalización al asignar proveedor
proveedor=info.get('proveedor', 'DESCONOCIDO')

# DESPUÉS: Se normaliza desde el inicio
proveedor_raw = info.get('proveedor', 'DESCONOCIDO')
proveedor_normalizado = normalizar_proveedor(proveedor_raw)
```

### 2. Función `normalizar_proveedor()` mejorada

- Busca alias ANTES de limpiar (captura `2T25 0422 CERES` → `CERES`)
- Quita prefijos: `1T25 0422`, `ATRASADA`, números de gestoría
- Quita sufijos: `OJO RETENCION`, `SIN APUNTE`, `TF`, `RC`, ` 2`, ` 3`
- Corrige typos por similitud

### 3. 290 alias en ALIAS_DICCIONARIO

### 4. Nuevo diccionario `CATEGORIAS_FIJAS_PROVEEDOR`

Para proveedores que SIEMPRE tienen la misma categoría (sin extractor específico):

```python
CATEGORIAS_FIJAS_PROVEEDOR = {
    'LUCERA': 'ELECTRICIDAD LOCAL',
    'ENERGIA COLECTIVA': 'ELECTRICIDAD LOCAL',
    'SOM ENERGIA': 'ELECTRICIDAD LOCAL',
    'YOIGO': 'TELEFONO',
    'WEBEMPRESA': 'WEB Y SOFTWARE',
    'ANTHROPIC': 'WEB Y SOFTWARE',
    ...
}
```

### 5. Soporte para `importe_iva_inc` (BM y similares)

Para extractores donde los importes INCLUYEN IVA:
- El extractor devuelve `importe_iva_inc` (NO `base`)
- main.py busca IVA y categoría en el diccionario
- Calcula `base = importe_iva_inc / (1 + iva/100)`
- Si no encuentra: `iva=0`, `categoria='PENDIENTE'`
